# Ben's Shed — Brand Guide

---

## 1. Brand Summary

**Ben's Shed** is a private, invite-only stock value scoring app used by a small group of friends and family. It pulls financial data, runs it through a set of homemade heuristics, and spits out scores that are — by design — vibe-coded nonsense dressed up in a spreadsheet's clothing.

The name says it all. This isn't a Bloomberg terminal. It's a shed. Someone built it on weekends. The tools inside are functional, occasionally clever, and held together with enthusiasm rather than rigour. It knows what it is.

**Core identity:**

- Private workshop, not a product launch.
- Self-aware about the limits of its own scoring — and says so, plainly.
- Treats the user as a friend who's been let in, not a customer being onboarded.
- Nerdy and a bit dry, never loud or try-hard.
- The humour is in the understatement, not in the punchlines.

**What it is not:**

- A fintech startup pretending to have an edge.
- An "AI-powered" anything.
- A joke app that doesn't actually work.

It works fine. It's just not to be trusted with your pension.

## 2. Three Brand Directions

---

### Direction A: "The Workshop Manual"

**Brand essence:**
Ben's Shed looks and feels like a well-organised workshop manual — clear labels, no-nonsense layout, quiet competence. The personality comes through in small moments of dry commentary tucked into otherwise functional copy. It takes its *structure* seriously but not its *conclusions*.

**Tone of voice:**

- Write like a technical manual that occasionally breaks character.
- Default to plain, direct language. No filler.
- Humour is earned: one dry aside per screen, maximum.
- Never explain the joke. If you have to flag that something is funny, cut it.
- Treat the user as someone who already gets it.

**Visual identity:**

- **Colours:** Off-white background (`#F5F3EF`, warm paper), near-black text (`#1A1A1A`), muted olive accent (`#6B7F5E`) for interactive elements, faded red (`#B5543A`) for warnings/errors. No bright colours.
- **Typography:** Monospace for the wordmark and data (e.g., JetBrains Mono, IBM Plex Mono). A clean sans-serif for body text (e.g., Inter, IBM Plex Sans). Nothing decorative.
- **UI feel:** Dense, tabular, slightly mechanical. Thin borders, minimal padding. Feels like a well-formatted config file or a workshop logbook. No cards with drop shadows. No rounded corners on everything.

**Logo/wordmark idea:**
"BEN'S SHED" in uppercase monospace, tracked out slightly. A small wrench or spanner glyph sits where the apostrophe would be, used sparingly. The wordmark looks like it was typed, not designed.

**Example microcopy:**

| Element | Copy |
|---|---|
| **App title + tagline** | **Ben's Shed** — Stock scores, loosely held. |
| **Login heading** | Come in. |
| **Login subcopy** | It's not locked. You just need the code. |
| **Section: Scoring** | The Numbers |
| **Section: Research** | Background Reading |
| **Empty state (no stocks)** | Nothing here yet. Add a ticker and we'll run it through the machine. |
| **Disclaimer** | These scores are the output of one person's spreadsheet logic and an unreasonable amount of confidence. They are not financial advice. They are barely financial *observation*. Please do not make investment decisions based on what you find here. If you do, that's on you. |

---

### Direction B: "The Shed Door"

**Brand essence:**
Ben's Shed leans into the literal shed metaphor — a warm, slightly cluttered personal space where projects live in various states of completion. The brand feels like a hand-painted sign on a garden shed door. Approachable, homemade, and completely unbothered by what the serious people are doing.

**Tone of voice:**

- Conversational, as if explaining something to a friend who's just walked in.
- Self-deprecating about the app's authority, never about its usefulness.
- Comfortable with silence — not every element needs clever copy.
- British understatement over American enthusiasm.
- Abbreviations and informality are fine. Full sentences optional.

**Visual identity:**

- **Colours:** Warm cream background (`#FAF7F2`), charcoal text (`#2C2C2C`), dusty blue accent (`#5B7B8A`) for links and buttons, soft amber (`#C49A3C`) for highlights/tags. Palette feels like old wood and afternoon light.
- **Typography:** A slightly characterful sans-serif for headings (e.g., Nunito, Quicksand — something with a bit of roundness but not childish). System sans-serif for body. Monospace only for stock tickers and numbers.
- **UI feel:** Comfortable spacing, warm tones. Feels like a personal wiki or a well-kept Notion page. Not polished, but cared for. Borders are soft or absent. Background textures optional (subtle paper grain).

**Logo/wordmark idea:**
"Ben's Shed" in a friendly, slightly hand-drawn sans-serif. Lowercase. A tiny roof peak (like `^` or a chevron) sits over the "h" in Shed, suggesting a shed silhouette without being literal. Simple enough to work at 16px.

**Example microcopy:**

| Element | Copy |
|---|---|
| **App title + tagline** | **Ben's Shed** — A place for stock ideas and rough guesses. |
| **Login heading** | Shed's open. |
| **Login subcopy** | Pop in. You know the code. |
| **Section: Scoring** | Score Board |
| **Section: Research** | The Corkboard |
| **Empty state (no stocks)** | Quiet in here. Search for a stock to get started. |
| **Disclaimer** | This is a hobby project. The scores are made up — well, *computed*, but from rules that one person thought sounded right. None of this is financial advice. Invest based on this and you're essentially trusting a shed. |

### Direction C: "The Back of the Envelope"

**Brand essence:**
Ben's Shed presents itself as the digital equivalent of working something out on the back of an envelope — quick, scrappy, probably directionally right, definitely not peer-reviewed. The brand is intellectual but unpretentious, like a napkin sketch that turned out to be interesting enough to keep.

**Tone of voice:**

- Academic-casual. Like a lecturer who's gone off-syllabus.
- Comfortable with qualifiers: "probably", "roughly", "give or take."
- Openly flags where its own logic is weakest.
- Dry, not playful. The humour is structural (e.g., a disclaimer that's longer than the feature it disclaims).
- No exclamation marks. Ever.

**Visual identity:**

- **Colours:** Pure white background (`#FFFFFF`), very dark grey text (`#1C1C1C`), muted indigo accent (`#4A5568`) for interactive elements, soft coral (`#D4725B`) for alerts. Sparse use of colour — mostly a black-and-white reading experience with occasional highlights.
- **Typography:** Serif for headings and the wordmark (e.g., Literata, Source Serif Pro, Lora). Clean sans-serif for body and UI (e.g., Inter). Monospace for data. The serif gives it the feel of a working paper or personal essay.
- **UI feel:** Generous whitespace, left-aligned, long lines. Feels like a well-typeset blog post or an academic preprint. Tables are simple, with thin horizontal rules only. Minimal chrome. The data does the talking.

**Logo/wordmark idea:**
"Ben's Shed" in a serif typeface, title case, no icon. A thin horizontal rule underneath. Looks like a chapter heading in a book nobody asked to be published. Alternatively, a small "[draft]" label in grey sits permanently next to the wordmark.

**Example microcopy:**

| Element | Copy |
|---|---|
| **App title + tagline** | **Ben's Shed** — Rough scores for patient people. |
| **Login heading** | Sign in |
| **Login subcopy** | Private. Not important enough to hack. |
| **Section: Scoring** | Value Scores |
| **Section: Research** | Notes & Sources |
| **Empty state (no stocks)** | No stocks scored yet. Add one and we'll do some dubious arithmetic. |
| **Disclaimer** | The scoring methodology behind this app has not been backtested, peer-reviewed, or endorsed by anyone with a relevant qualification. It is, at best, a structured opinion. Treat it accordingly. Nothing here constitutes financial advice or a recommendation to buy, sell, or hold any security. |

## 3. Recommended Direction

### Direction A: "The Workshop Manual" — strongest pick.

**Why this one:**

**It ages the best.** Directions B and C both have strong personalities, but they lean on tone in a way that risks wearing thin. "The Shed Door" (B) is warm and charming on first visit but starts to feel cutesy after the 50th login. "Back of the Envelope" (C) is elegant but almost too restrained — it undersells the fact that the app is actually functional and useful day-to-day.

"The Workshop Manual" (A) strikes the right balance for a tool you'll use repeatedly:

- **Functional first, personality second.** The dry humour is in the margins, not the main event. The app feels like it works because it *looks* like it works — clean structure, clear labels, data-forward layout.
- **The monospace/tabular aesthetic fits the content.** You're looking at stock tickers, scores, and numbers. A mechanical, slightly dense UI is native to that data. It doesn't need to be dressed up.
- **The disclaimer lands hardest.** Direction A's "barely financial *observation*" line hits the exact register — dry, self-aware, clearly not legal boilerplate, but also not trying to be a comedy bit.
- **Easiest to maintain.** No textures, no hand-drawn elements, no serif typography to finesse. Monospace + sans-serif + muted palette is the simplest to implement and the hardest to get wrong in Streamlit.
- **Scales if you add features.** The "manual" frame absorbs new sections naturally — you just add another chapter. The shed metaphor in B gets strained when you add an AI chat feature to a "corkboard."

If you want to borrow from the others: take Direction C's disclaimer format (the longer, more explicitly hedged version) and Direction B's login warmth ("Shed's open" is better than "Come in" — consider using it).

### 4.1 App Name & Tagline

- **App name:** Ben's Shed
- **Tagline:** Stock scores, loosely held.
- **Usage rules:**
  - Always written as "Ben's Shed" — capital B, capital S, curly apostrophe where supported.
  - The tagline always appears in sentence case, with a full stop.
  - Never abbreviate to "BS" (for obvious reasons — though if someone notices, that's fine).
  - In page titles and metadata: "Ben's Shed" (no tagline).
  - In the header/nav bar: "Ben's Shed" as wordmark, tagline optional below on landing/login only.

### 4.6 Footer & Disclaimer Language

**Short disclaimer** (appears in the app footer on every page):

> Not financial advice. Not even close.

**Full disclaimer** (appears on an About page or expandable section, and on first login):

> The scores in this app are the output of one person's spreadsheet logic and a set of heuristics that have never been backtested, peer-reviewed, or stress-tested against reality. They are not financial advice. They are not a recommendation to buy, sell, or hold any security. They are, at best, a structured opinion — and at worst, confidently wrong.
>
> If you make investment decisions based on what you find here, that is entirely your responsibility. The person who built this app uses it too, which should tell you something about its risk profile.

**Data attribution line** (small, in footer or data sections):

> Market data via Yahoo Finance. Scores via guesswork.

### 4.7 Where to Lean Into "Bullshit, Vibe-Coded" — and Where Not To

This is the most important section. The app's self-awareness is its best feature, but it has to be handled carefully or the whole thing becomes a joke that nobody trusts enough to actually use.

**DO lean in:**

- **Disclaimers and footers.** This is where the self-deprecation lives. The disclaimer is the app's chance to say "we know" without undermining the main experience.
- **Empty states.** These are low-stakes moments where a bit of personality makes the app feel alive.
- **Scoring methodology descriptions.** If you ever expose how a score is calculated, be transparent and slightly self-mocking: "This weights trailing P/E at 30%, which felt right at the time."
- **About / info page.** Go to town. Explain the project's origin, why the scoring is what it is, who it's for.
- **Tooltips and help text.** Small, tucked-away moments: "This number means roughly what you think it means."
- **Error states and edge cases.** "Can't pull data for this one. Might be delisted, might be our fault."

**DO NOT lean in:**

- **Navigation and primary labels.** Keep these plain and functional. "Scores" not "The Vibes."
- **Data display.** When showing actual stock data, prices, or scores — play it straight. The numbers should look trustworthy even if the methodology isn't.
- **Login flow.** Light touch only. "Shed's open" is enough. Don't turn the login into a comedy sketch.
- **Action buttons.** "Add to watchlist" not "Yeet into watchlist." Keep actions clear and predictable.
- **Loading states beyond login.** Just show a spinner or skeleton. Don't narrate.

**The rule of thumb:** The *frame* around the data can be playful. The *data itself* should be presented straight. A user should be able to glance at a score, a chart, or a table and read it instantly without decoding irony.

|---|---|---|---|

| Headings | JetBrains Mono | monospace | 400 (Regular) |
| Body text | Inter | system-ui, sans-serif | 400 (Regular) |
| Labels / small text | Inter | system-ui, sans-serif | 500 (Medium) |
| Data / numbers / tickers | JetBrains Mono | monospace | 400 (Regular) |

**Streamlit-specific guidance:**

- Set page config: `st.set_page_config(page_title="Ben's Shed", page_icon="🔧", layout="wide")`
- Use custom CSS via `st.markdown()` to override Streamlit defaults for background colour, font, and link colour.
- Sidebar background should use the Surface colour (`#EDEAE4`).
- Avoid Streamlit's default blue. Override `primaryColor` in `.streamlit/config.toml`:
  ```toml
  [theme]
  primaryColor = "#6B7F5E"
  backgroundColor = "#F5F3EF"
  secondaryBackgroundColor = "#EDEAE4"
  textColor = "#1A1A1A"
  font = "sans serif"
  ```
- For the monospace headings, inject via CSS since Streamlit's theme config doesn't support per-element fonts.
- Data tables and metrics should use monospace. Streamlit's `st.dataframe` and `st.metric` can be lightly styled with CSS overrides.

### 4.10 Quick Reference: Voice Cheat Sheet

For anyone writing copy in this app:

- **Default register:** Calm, direct, slightly dry. Think "competent friend," not "brand voice."
- **Capitalisation:** Sentence case everywhere. No Title Case Headings.
- **Punctuation:** Full stops on sentences. No exclamation marks. Em dashes welcome.
- **Pronouns:** "You" and "we" (informal we — it's still one person's shed). Avoid "the user" or "one."
- **Length:** Short. If a label can be one word, use one word. If a description can be one sentence, use one sentence.
- **Humour frequency:** One dry moment per screen, maximum. Zero is also fine. The app should be *usable* first, *charming* second.
- **Words to avoid:** Alpha, edge, insights, precision, institutional, AI-powered, smart, next-generation, analytics engine, portfolio optimisation, data-driven. Basically anything that could appear in a YC pitch deck.
- **Words that fit:** Scores, guesses, heuristics, rough, probably, roughly, seems, looks like, for what it's worth, take with salt.
